# 需求文档

## 1. 应用概述

- **应用名称**：倒计时工具
- **应用描述**：一款简洁美观的倒计时工具，支持自定义倒计时时长，提供开始、暂停、重置操作，并通过本地存储记录每日完成的倒计时次数。

---

## 2. 页面结构与核心功能

### 2.1 页面总览

```
倒计时工具
├── 主操作页（倒计时设置与控制）
└── 每日统计页（当日及历史统计记录）
```

### 2.2 主操作页

**时间设置区**
- 默认倒计时时长为 1 小时（即 60 分钟 00 秒）
- 用户可在倒计时未开始或已重置状态下，通过输入框或数字滚轮分别设置小时、分钟、秒
- 设置范围：小时 0–23，分钟 0–59，秒 0–59；最小有效时长为 1 秒
- 时间以「HH:MM:SS」格式大字体居中展示，视觉突出

**倒计时控制区**
- 开始按钮：启动倒计时，按钮切换为「暂停」
- 暂停按钮：暂停倒计时，保留当前剩余时间，按钮切换为「继续」
- 继续按钮：从暂停处恢复倒计时
- 重置按钮：停止倒计时，将时间恢复为当前设置的初始时长

**倒计时完成提示**
- 倒计时归零时，页面给予明显的视觉反馈（如高亮闪烁或颜色变化）
- 同时触发浏览器通知或页面内提示音效（若浏览器权限允许）
- 自动将本次完成记录写入当日统计

### 2.3 每日统计页

- 展示当日日期及当日已完成的倒计时次数
- 以列表或卡片形式展示历史各日期的完成次数（按日期倒序排列）
- 数据来源于本地存储，页面加载时自动读取并渲染
- 提供「清空统计数据」操作，清空前需二次确认

---

## 3. 业务规则与逻辑

- 倒计时运行期间，时间设置区不可编辑，需重置后方可修改
- 每次倒计时自然归零（非手动重置）才计为一次「完成」，手动重置不计入统计
- 统计数据以日期为键存储于浏览器本地存储（localStorage），格式为：`{ \"YYYY-MM-DD\": 完成次数 }`
- 页面刷新或重新打开后，倒计时状态不保留（重置为默认时长），统计数据持久保留
- 日期以用户本地时间为准，跨零点后自动归入新的一天

---

## 4. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户设置时长为 0 | 禁止启动，提示「请设置有效的倒计时时长」 |
| 用户输入非数字或超出范围的值 | 自动修正为最近合法值，或清空并提示 |
| 浏览器不支持 localStorage | 页面内提示「当前环境不支持统计功能，数据将不被保存」，倒计时功能正常使用 |
| 用户关闭页面后重新打开 | 倒计时重置为默认时长，统计数据从本地存储恢复展示 |
| 清空统计数据时用户取消确认 | 不执行清空，数据保持不变 |

---

## 5. 验收标准

- 默认倒计时时长为 1 小时，时间展示格式为 HH:MM:SS
- 开始、暂停、继续、重置功能均可正常使用，状态切换准确
- 倒计时自然归零后，页面有明显视觉反馈，且当日统计次数 +1
- 统计数据写入 localStorage，刷新页面后数据不丢失
- 每日统计页可正确展示当日及历史各日期的完成次数
- 清空统计数据需二次确认，确认后数据清除，取消则保留
- 倒计时运行中，时间设置区不可编辑
- 设置无效时长（如全为 0）时，无法启动并有提示

---

## 6. 本期不实现功能

- 多个倒计时并行运行
- 倒计时模板保存与快速选择
- 云端数据同步或跨设备统计
- 自定义提示音上传
- 番茄钟或专注模式等扩展玩法
- 统计数据导出（CSV / Excel 等）
